# 变更管理知识库（ECN / DCN）

> 本文件供 Agent 推理使用，内化后落到用户情境，不原样整段丢给用户。

## 1. 概念边界

| 概念 | 含义 | 是否改产品技术状态 | 典型触发 |
|------|------|------------------|----------|
| **ECN**（Engineering Change Notice，工程变更通知） | 对产品技术状态的正式变更 | 是 | 设计优化、材料替换、工艺参数、BOM、图纸、工装变更 |
| **DCN**（Document Change Notice，文件变更通知） | 对文件的正式变更，不改产品实物 | 否 | SOP/WI、检验规范、体系文件、记录表单、作业标准更新 |
| **CPM**（变化点管理，见 N11） | 变更在现场发生时的受控记录 | —（执行侧） | 现场人/机/料/法/环变化点的识别-标识-验证-追溯 |
| **供应商变更管理**（见参考技能） | 供应商入向变更评估 | — | 供应商 4M1E 变更、PPAP 变更 |

**上下游关系**：ECN/DCN 是"授权变更"的受控流程；CPM 是变更批准后在现场的执行记录；供应商变更常需先过本企业的 ECN 评审。

## 2. ECN vs DCN 判别规则

- 变更对象带**零件号/图号/版本**，且影响产品实物或技术状态 → **ECN**。
- 变更对象是一份**文件**（编号/版本，无零件技术状态变化）→ **DCN**。
- 模糊时先问："这次变更改的是产品本身，还是只是描述产品的文件？"改产品=ECN，改文件=DCN。

## 3. 影响分析维度（通用参考，企业准则「待企业补充」）

- **4M 影响**：人/机/料/法/环哪些要素随之变化（含相邻工序）。
- **成本**：单件成本、模具/工装投入、验证费用。
- **交期**：验证周期、客户交期风险。
- **法规/CSR**：是否满足客户特殊要求、RoHS/REACH、安规、行业强制标准。
- **关联产品**：共用零件/工序的其他产品是否连带影响。
- **关联文件清单**（ECN 必查）：BOM、DFMEA/PFMEA、控制计划(CP)、作业指导书(SOP/WI)、检验规范、图纸、包装规范。
- **DCN 影响**：受影响岗位/工序、是否需要重新培训、关联文件是否同步。

## 4. 评审与批准路由（通用参考）

- ECN 通常需**跨职能会签**：设计 / 工艺 / 质量 / 采购 / 生产，最终由授权人批准。
- DCN 通常由**编制→审核→文控(DCC)批准发放**；文控负责受控分发与旧版回收。
- 批准层级依变更风险与企业授权矩阵而定，技能不替用户判定谁有批准权。

## 5. 切换与实施（phase-in / phase-out）

- 生效时点、旧版库存/在制品处置、新旧断点（批次号/序列号/日期）。
- 原则：先验证、后切换；未验证不得放行（与 N11 状态"待验证=禁止放行"一致）。

## 6. 标准条款索引（供引用，非强制套用）

- ISO 9001:2015 **8.3.6** 设计和开发更改（设计变更控制）。
- ISO 9001:2015 **8.5.6** 生产和服务提供的更改（生产变更控制）。
- IATF 16949 **8.3.6 / 8.5.6** 变更管理（含顾客要求、PPAP、可追溯）。
- 企业自有 PLM/ERP/文控(DCC)流程优先于本技能。

## 7. 常见坑（引导用户时提示）

- 只改了主文件，漏同步 FMEA/CP/检验规范 → 现场执行与文件"两张皮"。
- 验证未完成就切换生产 → 客户端暴露批量问题。
- 编号/版本混乱，变更不可追溯 → 发生不良无法定位是否由变更引起。
- 审批越级或缺会签 → 责任悬空，复盘推诿。
- 供应商变更未走本企业 ECN 评审直接采用 → 失控。
