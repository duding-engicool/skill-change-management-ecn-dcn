# 变更管理（change-management-ecn-dcn）

面向变更工程师、项目经理与文控(DCC)的变更管理（ECN/DCN）技能。对工程变更与文件变更做影响分析、评审路由、验证与切换计划，生成受控变更记录。覆盖 ECN（改产品技术状态）与 DCN（只改文件），与变化点管理(CPM)、供应商变更管理上下游互补。

## 适用岗位
- 变更工程师 / 项目经理：变更发起与归口，组织影响分析、串联评审、推动闭环。
- 设计 / 工艺工程师（PE）：判定技术可行性与验证准则，负责文件编制。
- 质量工程师（QE）：评估质量/合规影响，参与会签与验证确认。
- 文控（DCC）：DCN 审核、受控发放、旧版回收与分发。

## 解决的核心痛点
- 漏评关联影响：只改主文件，漏同步 FMEA/控制计划/检验规范，现场"两张皮"。
- 未验证先切换：变了没做首件/首批/型式验证，问题在客户端暴露。
- 编号混乱不可追溯：发生不良时无法定位是否由变更引起。
- 审批越级/缺会签：责任悬空，复盘推诿。
- 供应商变更失控：未走本企业 ECN 评审直接采用。

## 产出物
- `ECN变更记录_<编号>.txt`：纯文字版工程变更记录，适合评审流转/打印。
- `ECN变更记录_<编号>.md`：结构化 Markdown 工程变更记录。
- `DCN变更记录_<编号>.txt` / `.md`：文件变更记录（同上格式）。

## 快速开始
```bash
# 直接运行，使用内置 ECN 小样本产出样例 TXT+MD（写到当前目录）
python scripts/build_report.py

# 用内置 DCN 小样本
python scripts/build_report.py --type dcn

# 基于自有 JSON 数据生成
python scripts/build_report.py --input 你的数据.json --out-dir 你的目录

# 只生成纯文字版 / 只生成 Markdown
python scripts/build_report.py --format txt
python scripts/build_report.py --format md
```

## ECN vs DCN 判别
- **ECN（工程变更）**：改产品技术状态——设计/BOM/材料/工艺/图纸/工装，通常联动 FMEA/PPAP/控制计划。
- **DCN（文件变更）**：只改文件——SOP/WI/检验规范/体系文件/记录表单，不改产品实物。

## 影响分析维度（ECN）
4M 影响 / 成本 / 交期 / 法规·CSR / 关联产品；**关联文件清单**（BOM、DFMEA/PFMEA、控制计划、SOP/WI、检验规范、图纸）。

## 联动技能（纯提示）
- 变更已在现场发生：`change-point-management`（N11）
- 供应商入向变更：`供应商变更管理`（参考技能）
- FMEA / PPAP / 控制计划更新：对应技能
- 变更作为纠正措施：`8d-problem-solving`

## 说明
- 所有文档为简体中文；影响分析与评审路由为通用参考，企业具体准则「待企业补充」。
- 本技能只产出受控记录文档，不替代 PLM/ERP/文控系统流程；不替用户定批准/放行结论。
